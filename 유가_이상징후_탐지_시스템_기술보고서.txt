================================================================
     유가 이상징후 탐지 시스템 기술 도입 보고서
================================================================


1. 시스템 전체 구조
----------------------------------------------------------------
풀스택 단일 서버 구조 — 프론트엔드와 백엔드를 하나의 Node.js
프로세스에서 실행합니다.

도입 이유:
내부 업무용 시스템으로 복잡한 마이크로서비스 분리 없이
개발·배포·유지보수를 단순화하기 위함.


2. 핵심 기술 스택
----------------------------------------------------------------
구분             도입 기술                      선택 이유
---------------------------------------------------------------------------
프론트엔드       React + TypeScript + Vite      컴포넌트 재사용성, 타입 안전성, 빠른 빌드
백엔드           Node.js + Express              JS 풀스택 단일 언어, REST API 구현 용이
데이터베이스     PostgreSQL + Drizzle ORM       관계형 데이터 구조, 타입스크립트 타입 자동 생성
UI 컴포넌트      shadcn/ui + Tailwind CSS       일관된 디자인 시스템, 커스터마이징 자유도
서버 상태 관리   TanStack Query                 자동 캐싱·갱신, 로딩/에러 상태 관리 내장
인증             express-session + bcrypt       세션 기반 단순 인증, 비밀번호 해시 암호화


3. 데이터 수집 자동화
----------------------------------------------------------------
도입 기술: Node-cron + Puppeteer (Chromium 헤드리스 브라우저)

구현 내용:
- 매일 09:10(확정) · 16:10(잠정) KST에 오피넷(OPINET) CSV 자동 다운로드
- JavaScript가 필요한 동적 페이지이므로 단순 HTTP 요청으로는
  데이터 수집 불가 → 브라우저 자동화 필요

도입 이유:
오피넷 공식 API는 제공 범위가 제한적이고, 전국 주유소 가격 CSV는
로그인 후 동적으로 생성되는 구조여서 실제 브라우저 동작이 필요.


4. 외부 실시간 데이터 연동
----------------------------------------------------------------
데이터         출처                    방식
-----------------------------------------------------------
WTI 국제유가   Yahoo Finance           공개 API JSON 파싱
달러 환율      Open Exchange Rates     REST API
날씨           OpenWeatherMap          REST API


5. 이상징후 탐지 로직
----------------------------------------------------------------
수집된 데이터에서 전일 대비 100원 이상 변동 및 최고가격제 상한 초과
주유소를 자동 산출하여 대시보드에 리포트로 표시.

별도 AI 외부 API 없이 순수 데이터 비교 연산으로 구현.


6. 알림 (푸시 알림)
----------------------------------------------------------------
도입 기술: Web Push (PWA 표준) + Service Worker

도입 이유:
앱스토어 배포 없이 모바일 홈화면 추가만으로 네이티브 앱 수준의
푸시 알림 수신 가능.
데이터 수집 완료 시 MASTER 계정에 결과 자동 발송.


7. 권한 체계
----------------------------------------------------------------
HQ_USER : 대시보드 + 유가 분석 조회
MASTER  : 전체 관리 (사용자·본부·팀 관리, 최고가격제 설정)

본부별 지역 데이터 범위를 DB 레벨에서 분리하여
담당 지역 외 데이터 접근 차단.


8. 배포
----------------------------------------------------------------
VM (Always-on) 방식 채택

이유:
오토스케일 방식은 서버가 꺼졌다 켜지므로 크론 스케줄러가 중단됨.
매일 정해진 시간에 데이터 수집이 필수인 이 시스템은 항상 켜져
있어야 하므로 VM 배포 선택.
