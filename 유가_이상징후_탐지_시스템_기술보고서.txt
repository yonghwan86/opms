1. 시스템 개요 및 목적
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

시스템명    : OPMS (Oil Price Monitoring System) — 유가 모니터링 상황판
배포 주소   : opms.kpetro.or.kr
운영 주체   : 한국석유공사 관련 조직

[목적]
전국 주유소의 일별 소매 유가를 자동으로 수집·분석하여 가격 이상징후
(비정상적 급등·급락, 최고가격제 위반 의심 등)를 조기에 탐지하고,
국제 원유·석유제품 가격 및 정유사 공급가 흐름과 함께 종합 모니터링
하는 업무 지원 시스템입니다.

[핵심 기능 요약]
  ① 전국 주유소 소매 유가 자동 수집 (1일 2회)
  ② 국제 원유(WTI·브렌트·두바이) 및 석유제품 가격 자동 수집
  ③ 정유사 4사 주간 공급가격 자동 수집
  ④ 유가 이상징후 탐지 및 TOP 10 리포트 제공
  ⑤ 최고가격제 대비 실판매가 추이 시각화
  ⑥ 조직별 관할 지역 데이터 접근 권한 제어
  ⑦ PWA 푸시 알림 (데이터 갱신·이상징후 발생 시)

프론트엔드 정적 파일도 같은 서버에서 제공합니다.

4. 데이터 수집 자동화 — 5가지 수집 파이프라인
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

시스템은 총 5가지 종류의 데이터를 자동으로 수집합니다.

────────────────────────────────────────────────────────────
4-1. 국내 소매 유가 — 오피넷(OPINET) CSV 자동 수집
────────────────────────────────────────────────────────────
  수집 주기  : 매일 오전 09:30 (확정값) · 오후 16:30 (잠정값) KST
  데이터 출처: 한국석유공사 오피넷 (www.opinet.co.kr)
  수집 방법  : Playwright Chromium 헤드리스 브라우저 자동화

  [왜 일반 HTTP 요청이 아닌 브라우저 자동화가 필요한가?]
  오피넷 전국 주유소 가격 CSV는 로그인 후 NetFunnel이라는 대기열
  시스템을 통과해야 다운로드되는 구조입니다. JavaScript가 없으면
  페이지 자체가 작동하지 않으므로, 실제 브라우저를 자동으로 조작해
  파일을 받아오는 방식을 사용합니다.

  [수집 흐름]
  ① Playwright로 오피넷 접속
  ② NetFunnel B7 토큰 자동 처리 (자동화 차단 우회)
  ③ fn_Download(6) 함수 직접 호출 → CSV 파일 다운로드
  ④ EUC-KR 인코딩 → UTF-8 변환 (iconv-lite)
  ⑤ 파싱 및 정제 (가격 0원 = 미판매로 처리)
  ⑥ 전국 주유소별 일별 가격 DB 저장

  [오전·오후 이중 수집 이유]
  오피넷은 당일 최종 가격을 하루에 걸쳐 점진적으로 확정합니다.
  오전 09:30에 전날 확정값을 먼저 저장하고,
  오후 16:30에는 오늘 잠정값을 추가로 수집합니다.

────────────────────────────────────────────────────────────
4-2. 국내 평균 유가 — 오피넷 API 실시간 조회
────────────────────────────────────────────────────────────
  수집 주기  : 매일 09:00 · 12:00 · 16:00 · 19:00 KST (하루 4회) + 서버 시작 직후
  데이터 출처: 오피넷 공개 API
  수집 방법  : REST API 호출 (JSON 응답)

  전국 평균 휘발유·경유·등유 가격과 전일 대비 변동값을 가져와
  대시보드 상단 카드에 실시간으로 표시합니다.
  메모리 캐시에 저장하여 사용자 요청 시 즉시 응답합니다.

────────────────────────────────────────────────────────────
4-3. 국제 원유·석유제품 가격 — Petronet 크롤링
────────────────────────────────────────────────────────────
  수집 주기  : 화요일~토요일 오전 08:30 KST (Petronet 업데이트 요일 기준)
  데이터 출처: 한국석유공사 Petronet (www.petronet.co.kr)
  수집 방법  : HTML 파싱 크롤링 (Axios + HTML 분석)

  수집 항목:
  - 국제 원유가  : WTI, 브렌트(Brent), 두바이(Dubai) — 달러/배럴
  - 국제 석유제품: 국제 휘발유·경유·등유 — 달러/배럴
  - 전일 대비 변동값

  [수집 결과 활용]
  대시보드에서 국제 원유가 시계열 차트와 국내 유가 연동 분석 차트로
  시각화됩니다. 국제 가격 상승이 국내 가격에 반영되는 시차를 확인할 수
  있습니다.

────────────────────────────────────────────────────────────
4-4. 정유사 주간 공급가격 — 오피넷 스크래핑
────────────────────────────────────────────────────────────
  수집 주기  : 매주 금요일 13:00 KST (공휴일인 경우 월요일 13:00)
  데이터 출처: 오피넷 주간 공급가격 페이지
  수집 방법  : Axios + HTML 테이블 파싱

  수집 항목:
  - SK에너지, GS칼텍스, HD현대오일뱅크, S-OIL 4사별
  - 고급 휘발유·휘발유·경유·등유 주간 공급가격

  [왜 공급가가 중요한가?]
  정유사 → 주유소 간 공급가격은 소매 유가 변동의 선행지표입니다.
  공급가가 오르면 약 1~2주 후 소매 유가도 오르는 경향이 있어,
  이상징후 조기 탐지에 활용됩니다.

────────────────────────────────────────────────────────────
4-5. 외부 실시간 지표 — API 연동
────────────────────────────────────────────────────────────

  원/달러 환율: Open Exchange Rates API

  국제 유가 변동이 환율을 통해 국내 가격에 영향을 미치는 흐름을
  함께 볼 수 있도록 제공됩니다.

5. 수집 장애 대응 — 자동 복구 및 재시도 체계
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[문제 상황]
서버는 간헐적으로 재시작됩니다 (인프라 유지보수, 업데이트 등).
서버가 정기 수집 시간(08:30, 09:30 등) 이후에 재시작되면
그 시간의 수집이 자동으로 누락될 수 있습니다.

[대응 방법 1 — 자동 재시도]
수집이 실패하거나 데이터가 없는 경우, 일정 시간 후 자동으로
다시 시도합니다.

  수집 종류              재시도 횟수   재시도 간격
  ─────────────────────────────────────────────────
  국내 소매 유가          최대 2회     10분 후, 20분 후
  국제 원유·석유제품      최대 2회     1시간 후, 2시간 후
  정유사 주간 공급가      최대 2회     1시간 후, 2시간 후

[대응 방법 2 — 시작 복구 (Startup Recovery)]
서버가 시작될 때 자동으로 수집 누락 여부를 확인하고,
필요하면 즉시 수집을 실행합니다.

  복구 대상          확인 조건
  ─────────────────────────────────────────────────────────────────
  국내 소매 유가     KST 09:30 이후 시작 + 오늘 오전 수집 기록 없음
                     → 즉시 오전 확정 수집 실행
  국제 유가         KST 08:30 이후 시작 + 화~토요일 +
                     최근 5일 이내 국제가격 데이터 없음
                     → 즉시 Petronet 크롤링 실행

[대응 방법 3 — 수집 이력 로그]
모든 자동·수동 수집은 결과(성공/실패/부분성공/건너뜀)를 DB에 기록합니다.
관리자는 "유가 수집 이력" 메뉴에서 날짜·유형·결과별로 조회하고
CSV로 다운로드할 수 있습니다.

  기록하는 항목: 수집 시각, 수집 유형, 결과, 대상 날짜, 수집 건수,
                 분석 건수, 소요 시간, 오류 내용

6. 유가 이상징후 탐지 로직
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

수집된 전국 주유소 데이터를 3가지 기준으로 분석하여
각 기준별 상위 10개 주유소를 추출합니다.

────────────────────────────────────────────────────────────
분석 유형 1 — MAX_MIN : 가격 극단값 탐지
────────────────────────────────────────────────────────────
  - 최고가 TOP 10: 특정 지역·브랜드에서 유독 비싼 주유소 식별
  - 최저가 TOP 10: 비정상적으로 낮은 가격 주유소 (투매·미판매 오류 가능)
  - 유종: 휘발유·경유·등유 각각 산출

────────────────────────────────────────────────────────────
분석 유형 2 — CHANGE : 전일 대비 가격 변동 탐지
────────────────────────────────────────────────────────────
  - 급등 TOP 10: 전날 대비 가장 많이 오른 주유소 (공급가 반영 또는 이상 징후)
  - 급락 TOP 10: 전날 대비 가장 많이 내린 주유소
  - 별도 AI 없이 순수 가격 차이 연산으로 구현

────────────────────────────────────────────────────────────
분석 유형 3 — DIFF : 휘발유-경유 가격 격차 탐지
────────────────────────────────────────────────────────────
  - 격차 최대 TOP 10: 동일 주유소 내 휘발유와 경유 가격 차이가 가장 큰 곳
  - 격차 최소 TOP 10: 차이가 비정상적으로 작은 곳
  - 가격 역전 현상(경유>휘발유)을 발견하는 데 유용

────────────────────────────────────────────────────────────
최고가격제 변동추이 (별도 분석 기능)
────────────────────────────────────────────────────────────
  정부가 고시한 최고가격 기준(상한가)과 실제 판매가를 주유소별로
  비교하여 초과 여부와 변동 추이를 시각화합니다.
  담당자가 관할 지역 주유소의 최고가격 준수 현황을 한눈에 파악할 수
  있습니다.

    임의로 우회할 수 없습니다.

11. 인증 (로그인) 흐름
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  [2단계 로그인 방식]

  단계 1: 아이디 입력
          → 서버에서 사용자 존재 여부·초기 설정 필요 여부 확인
          → 초기 비밀번호 설정이 필요한 신규 사용자는 설정 화면으로 이동

  단계 2: 비밀번호 입력
          → bcrypt 해시 비교 → 일치하면 서버 세션 발급
          → 이후 모든 API 요청에 쿠키 세션으로 인증

  [보안]
  - 비밀번호는 bcrypt 단방향 해시로만 저장 (원문 복원 불가)
  - 세션은 PostgreSQL에 영속 저장 (서버 재시작 후에도 로그인 유지)
  - 모든 API는 인증 미들웨어를 통과해야 접근 가능
  - 로그인 성공·실패 이력을 DB에 기록 (IP, 접속 기기 포함)

12. 배포 환경
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  플랫폼   : Replit (VM 상시 실행 방식)
  배포 주소 : opms.kpetro.or.kr
  DB       : Replit 내장 PostgreSQL
  환경변수  : DATABASE_URL, SESSION_SECRET,
             VAPID_PUBLIC_KEY, VAPID_PRIVATE_KEY

  [VM(Always-on) 방식 선택 이유]
  오토스케일(Autoscale) 방식은 요청이 없으면 서버가 자동으로 꺼집니다.
  이 시스템은 매일 정해진 시간에 반드시 자동 수집이 실행되어야 하므로,
  서버가 항상 켜져 있는 VM 방식을 선택했습니다.

  [인프라 재시작 대응]
  서버가 예기치 않게 재시작되더라도 시작 복구 로직(5장 참조)이
  자동으로 수집 누락을 감지하고 즉시 보완합니다.
