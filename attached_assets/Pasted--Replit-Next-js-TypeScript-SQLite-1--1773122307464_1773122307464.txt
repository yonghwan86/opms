너는 시니어 풀스택 개발자다.

Replit에서 바로 실행 가능한 Next.js + TypeScript + SQLite 기반 웹 시스템을 개발한다.

이번 1단계에서는 오피넷 API는 아직 붙이지 말고, 계정/본부/팀/사용자/지역권한 관리 시스템만 먼저 구현한다.

이 시스템은 향후 유가정보 수집·분석·푸시 시스템의 권한 기반 관리자 웹앱이다. 이번 단계의 목적은 조직 구조와 사용자 권한, 지역 접근 통제를 먼저 완성하는 것이다.

[핵심 운영 구조]
1. 모든 사용자는 개별 계정을 가진다.
2. 사용자는 반드시 하나의 본부와 하나의 팀에 소속된다.
3. 팀은 반드시 하나의 본부에 소속된다.
4. 부서는 조직 정보로만 저장한다. 권한 기준은 아니다.
5. 지역 권한은 "본부 단독" 또는 "팀 단독"이 아니라 "본부 + 팀 조합"에 할당된다.
6. 사용자는 자기 소속 본부 + 팀 조합에 할당된 지역 정보만 조회 가능해야 한다.
7. MASTER는 전체 본부/팀/사용자/지역을 조회·관리 가능하다.
8. 동일 계정의 중복 로그인 허용.
9. 여러 브라우저/기기 동시 로그인 허용.
10. 로그인 로그 및 감사 로그 저장.

[권한 구조]
1. MASTER
- 전체 본부 관리
- 전체 팀 관리
- 전체 사용자 관리
- 사용자 비밀번호 초기화
- 사용자 엑셀 업로드
- 본부+팀 조합별 지역 권한 할당
- 전체 로그인 로그 조회
- 전체 감사 로그 조회

2. HQ_USER
- 자기 계정 조회
- 자기 본부 정보 조회
- 자기 팀 정보 조회
- 자기 본부+팀 조합에 할당된 지역 목록 조회
- 다른 본부/다른 팀/허용되지 않은 지역 접근 시 403 반환

[엑셀 업로드 요구사항]
MASTER만 가능.
xlsx 파일 업로드 지원.

엑셀 컬럼은 반드시 아래 포함:
username
display_name
email
position_name
department_name
headquarters_code
team_code
role
enabled
initial_password

검증 조건:
- 파일 확장자 .xlsx 확인
- 필수 컬럼 누락 검사
- 빈 행 제거
- username 중복 검사 (DB + 파일 내부)
- email 중복 검사 (DB + 파일 내부)
- headquarters_code 존재 여부 확인
- team_code 존재 여부 확인
- team은 반드시 해당 headquarters에 속해야 함
- role은 MASTER 또는 HQ_USER만 허용
- initial_password 없으면 임시 비밀번호 자동 생성
- 오류 발생 시 해당 행 실패 처리
- 성공/실패 건수 및 실패 사유 반환
- 업로드 결과 audit_logs 기록

본부/팀 자동 생성 금지.
존재하지 않는 code는 오류 처리.

샘플 템플릿 다운로드 기능 포함:
헤더:
username,display_name,email,position_name,department_name,headquarters_code,team_code,role,enabled,initial_password

예시 데이터 3행도 포함해서 xlsx 파일로 다운로드 가능하게 구현하라.

[지역 권한 처리 방식]
- MASTER가 "본부 + 팀 조합" 기준으로 지역 권한을 등록/수정/삭제 가능
- HQ_USER가 조회 가능한 지역은 자기 headquarters_id + team_id 조합에 연결된 지역만 해당
- HQ_USER는 다른 본부의 팀 권한이나 같은 본부의 다른 팀 권한을 볼 수 없음
- 권한 없는 지역 접근 시 403 반환
- 권한 체크 로직은 공통 미들웨어 또는 공통 권한 유틸로 분리
