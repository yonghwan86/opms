Task 1 — 초기 동기화: 한 가지 보완이 필요합니다
"최근 날짜 기준으로 추출"이라고 했는데, 어떤 날짜 기준인지 명시하면 좋겠습니다. 최근 1일만 쓰면 당일 미판매 주유소가 누락됩니다. 최근 7일 정도의 DISTINCT station_id를 합산하는 게 안전합니다.
sqlINSERT INTO gas_stations_master (station_id, station_name, sido, region)
SELECT DISTINCT ON (station_id)
  station_id, station_name, sido, region
FROM oil_price_raw
WHERE date >= (SELECT MAX(date) FROM oil_price_raw)::text::int - 7
ORDER BY station_id, date DESC
ON CONFLICT (station_id) DO UPDATE SET
  station_name = EXCLUDED.station_name,
  sido = EXCLUDED.sido,
  region = EXCLUDED.region,
  updated_at = now();
그리고 10,521건이라고 숫자까지 적었는데, 이건 Dev DB 기준이겠죠? 운영 DB는 다를 수 있으므로 "Dev 기준 약 10,500건, 운영은 검증 필요" 정도로 적으면 좋겠습니다.
또한 gas_stations_master 스키마에 현재 brand 컬럼이 없습니다. searchStations의 반환값에 brand가 포함되어야 하는데, master에서 brand를 가져올 수 없으면 Step 2에서 oil_price_raw JOIN 시 가져와야 합니다. 이건 Task 4에서 처리되겠지만, Task 1의 동기화 시점에 brand도 master에 추가할지 말지 결정해두면 좋습니다. 추가하면 검색 결과 표시가 더 깔끔해지고, 추가하지 않으면 스키마 변경 없이 진행할 수 있습니다.

Task 2 — 자동 upsert: 위치 선택에 주의가 필요합니다
saveOilPriceRaw() 내부에 넣겠다고 했는데, 이 함수는 500건 단위 청크로 INSERT합니다:
typescriptasync saveOilPriceRaw(rows: InsertOilPriceRaw[]): Promise<void> {
  const CHUNK = 500;
  for (let i = 0; i < rows.length; i += CHUNK) {
    const chunk = rows.slice(i, i + CHUNK);
    await db.insert(oilPriceRaw).values(chunk).onConflictDoUpdate({...});
  }
}
여기에 master upsert를 넣으면 매 청크(500건)마다 master upsert가 실행됩니다. 전국 주유소 약 1만 건이면 20번 반복 × master upsert = 비효율적입니다.
더 나은 위치는 saveOilPriceRaw() 외부, 호출 직후입니다. 스케줄러에서 saveOilPriceRaw(insertRows) 호출 후 별도로 master 동기화를 실행하면 1회만 실행됩니다:
typescript// oilScheduler.ts — saveOilPriceRaw 호출 후
await storage.saveOilPriceRaw(insertRows);
await storage.syncGasStationsMaster(insertRows); // 신규 함수
또는 saveOilPriceRaw() 끝에 한 번만 실행하는 방식도 괜찮습니다:
typescriptasync saveOilPriceRaw(rows: InsertOilPriceRaw[]): Promise<void> {
  // ... 기존 청크 INSERT ...

  // 마지막에 master 동기화 (1회)
  const uniqueStations = new Map<string, InsertOilPriceRaw>();
  for (const r of rows) {
    uniqueStations.set(r.stationId, r);
  }
  // station_id 기준 중복 제거 후 벌크 upsert
}

### Task 4 — searchStations 개선: 방향은 맞지만 보완이 필요합니다

"week_supply_avg CTE는 조회된 날짜 범위로 범위를 제한"이라고 했는데, 이 부분을 좀 더 구체적으로 설계해야 합니다.

현재 `week_supply_avg`는 전체 `oil_weekly_supply_prices`를 GROUP BY하는데, 실제로 필요한 건 **최근 20일에 해당하는 주차**뿐입니다. Step 2에서 날짜 목록을 가져온 후 주차 키를 계산해서 WHERE 조건에 넣으면 됩니다.

그리고 Step 1에서 master로 station_id를 찾고 Step 2에서 `(station_id, date)` 인덱스로 가격을 가져오는 구조는 맞는데, 현재 쿼리가 반환하는 `brand`, `is_self`, `address` 등의 컬럼은 **master에 없고 oil_price_raw에만 있습니다.** Step 2의 JOIN에서 이 컬럼들을 oil_price_raw에서 가져와야 합니다. 이 점은 현재 설계로 커버되지만, Task 설명에 명시하면 구현 시 혼란이 없습니다.
